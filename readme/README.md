# 価格検索ツール 使い方

---

## ファイル構成

```
作業フォルダ/
├── price_db.db                   ← SQLiteデータベース（初回実行で自動生成）
├── sqlite_price_lookup.py        ← 価格検索本体（VBAから呼ばれる）
├── db_import.py                  ← CSVをDBに取込むスクリプト
├── run_import.bat                ← ダブルクリックで取込実行
├── 使い方.md
└── csv/                          ← CSVを置くフォルダ
    ├── 標準DB.csv
    ├── 見積DB.csv
    ├── エイブル価格表DB.csv
    └── 面積設定.csv              ← 面積別特価の閾値（任意。下記「面積による特価」参照）
```

---

## ファイルをローカルに保存する

このツールはローカルPC（Windows）上で動かします。GitHubから一式をダウンロードして、1つの作業フォルダにまとめてください。

### 方法A：ZIPでまとめてダウンロード（推奨・Git不要）

1. GitHubのリポジトリ画面で緑色の **「Code」** ボタンをクリック
2. **「Download ZIP」** を選ぶ
3. ダウンロードしたZIPを右クリック →「すべて展開」で解凍
4. 解凍したフォルダを、使いやすい場所（例: `デスクトップ\価格検索ツール`）に置く

これで「ファイル構成」のファイルが1つのフォルダに揃います。

### 方法B：Gitでクローン（Gitが使える人向け）

```
git clone https://github.com/goodconduct278/Price-search.git
```

最新版に更新したいときは、フォルダ内で次を実行します。

```
git pull
```

#### Pythonスクリプトをgitで更新する手順

`sqlite_price_lookup.py` / `db_import.py` を最新に更新する流れです。

1. **Excelを閉じる**（スクリプトが使用中だと更新できないことがあるため）
2. クローンしたフォルダで以下を実行

   ```
   cd 価格検索ツールのフォルダ
   git pull
   ```

3. ローカルでCSVやファイルを編集していて `git pull` が競合（conflict）した場合は、
   自分の変更を退避してから取得します。

   ```
   git stash          ← 自分の変更を一時退避
   git pull           ← 最新を取得
   git stash pop      ← 退避した変更を戻す（必要な場合）
   ```

   ※ `price_db.db` や `csv/` の中身はローカル管理なので、基本はリポジトリに含めず
   　自分のデータはそのまま残ります。

4. 更新後は **Excelを開き直し、`run_import.bat` で取込を実行**して反映します。
5. **VBAマクロも更新があった場合**は、`readme/PriceLookup.bas` をExcelに再インポート
   してください（古いモジュールは削除してから取り込む）。

> Gitを使わない場合は「方法A：ZIPでダウンロード」で最新ZIPを取り直し、
> Pythonスクリプトを上書きしてもOKです。

### 保存時の注意

- `sqlite_price_lookup.py` / `db_import.py` / `run_import.bat` / `価格検索ツール.xlsm` は **すべて同じフォルダ** に置くこと（VBAが同フォルダ前提でスクリプトを呼ぶため）。
- `price_db.db` は初回の取込で自動生成されるので、最初は無くてOK。
- ファイルを上書き保存・更新したら、Excelはいったん閉じてから開き直す。
- Pythonが未インストールの場合は、先に [python.org](https://www.python.org/) からインストールし、インストール時に **「Add python.exe to PATH」** にチェックを入れる（VBAは `py` コマンドでPythonを呼びます）。

## 初回セットアップ

1. `csv/` フォルダにCSVを入れる
2. `run_import.bat` をダブルクリック
3. VBAモジュール（.bas）をExcelにインポート

以上で完了。

---

## Excelでの使い方（日常）

1. B2セルで検索対象DBを選ぶ（例: `標準DB`）
2. **E2セルに面積(㎡)を入力**（面積別特価を使う場合のみ。不要なら空でOK）
3. B4以降に製品名を入力
4. マクロ `RunPriceLookupViaPython` を実行
5. 候補が複数ある場合はポップアップが出るので番号または品番CDを入力

**結果はC〜G列に入る**

| 列 | 内容 |
|----|------|
| C  | 価格（面積特価が適用された場合はその価格） |
| D  | 荷姿 / 単位 |
| E  | 結果メモ（参照DB・品番CD・適用区分・類似度等） |
| F  | 品番CD |
| G  | 正式材料名 |

> ※ パラメータ行（2行目）: **B2＝検索対象DB / E2＝面積(㎡)**。製品名は4行目以降。

## データを更新したいとき

1. `csv/` フォルダ内の該当CSVを直接編集
2. `run_import.bat` をダブルクリック

特定のソースだけ更新したい場合はコマンドで実行:

```
python db_import.py --db price_db.db --csvdir csv --source 標準DB
```

## 新しいDBを追加したいとき

1. `csv/` フォルダに `○○社DB.csv` を置く
2. `run_import.bat` をダブルクリック
3. Excelの B2セルに `○○社DB` と入力して使える

スクリプトの変更は不要。

## CSVのフォーマット

| 列名 | 必須 | 備考 |
|------|------|------|
| 品番CD（または 品番コード） | ○ | |
| 材料名（または 品番名） | ○ | |
| 価格 | - | なければ名称のみ取込 |
| 荷姿 / 単位 | - | あれば取込む |
| 備考 | - | あれば取込む |
| 特価1 | - | 面積別特価の1段目価格（下記参照） |
| 特価2 | - | 面積別特価の2段目価格（下記参照） |
| その他の列 | - | 無視される |

エンコード: UTF-8（BOMあり）またはCP932（Shift-JIS）

## 面積による特価（段階価格）

価格表によっては、**面積が大きいほど安い特価**が設定されている場合があります。
（例: 301㎡以上で特価1、701㎡以上で特価2）。設定が無い価格表は通常価格のままです。

### 仕組み

Excelの **E2セルに面積(㎡)** を入れてマクロを実行すると、面積に応じて価格が切り替わります。

| 入力した面積 | 適用される価格 |
|--------------|----------------|
| 特価1の面積 未満 | 通常の `価格` |
| 特価1の面積 以上 〜 特価2の面積 未満 | `特価1` |
| 特価2の面積 以上 | `特価2` |

- E2が空、またはそのDBに面積設定が無い場合は、面積を無視して通常価格になります。
- 特価1/特価2の列が空の商品は、面積を入れても通常価格のままです。

### 設定の入れ方

**① 価格表CSVに `特価1` / `特価2` の列を追加**（特価のある商品にだけ金額を入れる）

```
品番CD,材料名,価格,荷姿 / 単位,特価1,特価2
89,TFコート ライトグレー,12900,20kg／缶,11500,9800
100,通常品のみ,5000,缶,,
```

**② 面積の閾値は別ファイル `csv/面積設定.csv` で管理**（DBごとに異なる値を設定可）

```
DB名,特価1面積,特価2面積
エイブルDB,301,701
```

- `DB名` … 価格表CSVのファイル名（拡張子なし＝B2で選ぶDB名）と一致させる
- `特価1面積` … 特価1に切り替わる面積(㎡)。上の例では301㎡以上
- `特価2面積` … 特価2に切り替わる面積(㎡)。上の例では701㎡以上
- この `面積設定.csv` に**載っていないDBは面積処理なし**（通常価格のまま）

設定したら、いつもどおり `run_import.bat` をダブルクリックして取り込めば反映されます。

## 価格検索の優先順位

B2で選んだDBに価格がない場合、自動でフォールバックする。

```
B2で選んだDB → 標準DB → 見積DB
```

## よくあるエラー

| エラー | 原因 | 対処 |
|--------|------|------|
| `DBファイルが見つかりません` | dbPathとDB名が不一致 | VBAの `dbPath` を確認 |
| `Pythonスクリプトが見つかりません` | スクリプトが別フォルダにある | 同じフォルダに配置 |
| `対象行が多すぎます（XXXX行）` | B列に余分なデータが残っている | 該当セルを選択してDelete |
| `登録済みのDB: ...` エラー | B2のDB名がDBに存在しない | csv/にCSVを置いて取込を実行 |
| `候補なし` | 名称DBに未登録の製品名 | CSVに追記して再取込 |
